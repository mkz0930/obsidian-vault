## 议题一：MVP定义

### 核心公式

```
Apple Watch + Agent = 80%自研 + 10% OPPO
```

### 数据源

| 数据类型 | 来源 | 用途 |
|----------|------|------|
| 心率变异性 (HRV) | Apple Watch | 压力/恢复分析 |
| 睡眠分析 | Apple Watch | OSA风险评估 |
| 血氧饱和度 | Apple Watch | 呼吸监测 |
| 步数统计 | Apple Watch | 活动量评估 |

### 目标用户

- 有Apple Watch的用户
- 关注睡眠质量和呼吸健康
- 希望获得个性化健康建议

## 议题三：OSA睡眠呼吸暂停专项

### 项目背景

OSA (Obstructive Sleep Apnea) 是一种常见的睡眠呼吸障碍

### 核心功能

1. **数据采集**：从Apple Health导出睡眠数据
2. **模式识别**：分析睡眠阶段和呼吸事件
3. **风险评估**：基于HRV和血氧综合判断OSA风险
4. **建议生成**：提供改善睡眠的个性化建议

### 技术实现

| 模块 | 技术选型 | 状态 |
|------|----------|------|
| 数据解析 | Python + JSON | ✅ 已完成 |
| Agent框架 | LangChain ReAct | 🔄 开发中 |
| UI界面 | Gradio | 🔄 开发中 |

## 架构决策记录 (ADR)

### ADR 001：技术栈选型

**状态**：Accepted

**决定**：采用 LangChain + MiniMax API + Gradio 技术栈

**理由**：
- LangChain提供成熟的Agent框架
- MiniMax成本可控
- Gradio快速构建原型

**后果**：
- ✅ 开发效率高
- ⚠️ 需处理API限流问题
- ⚠️ 依赖第三方服务稳定性

---

### ADR 002：数据存储策略

**状态**：Accepted

**决定**：本地JSON存储 + 云端备份

**理由**：
- 隐私优先，数据本地化
- Apple Health导出为JSON格式
- 减少数据迁移成本
